¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Solicita una representación de un recurso en específico.
HEAD: Lo mismo que un GET, pero sin el cuerpo de la respuesta.
POST: Envío de datos al servidor.
PUT: Sustituye totalmente un recurso de OSLC.
PATCH: Aplica modificaciones parciales a un recurso.
DELETE: Borra un recurso

Métodos de petición HTTP - HTTP | MDN. (2022, November 29). Mozilla.org. https://developer.mozilla.org/es/docs/Web/HTTP/Methods

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No siempre. El error 404 significa NOT FOUND, puede significar que de verdad algo que se busca no está, por lo cual el desarrollador si debería hacerse cargo, pero en caso de que sea que se busca algo inexistente, entonces no es culpa del desarrollador.


¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

En este caso, el problema si debe reportarse con el desarrollador, ya que este problema es sobre un problema con el servidor, asunto que es de vital importancia corregir.
